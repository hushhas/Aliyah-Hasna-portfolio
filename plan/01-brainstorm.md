## Prompt 3
Lebih baik Single page atau multi-page?

## AI Response 3
🧭 Single Page vs Multi-page
✅ Single Page (Recommended)
Kenapa ini paling cocok:
1. Recruiter behavior itu cepat
HR biasanya cuma lihat:
10–30 detik pertama
Mereka tidak akan klik banyak halaman

👉 Single page = semua langsung terlihat dengan scroll

2. Lebih mobile-friendly (sesuai constraint kamu)
Scroll > klik page
UX lebih smooth di HP

👉 Ini penting karena banyak HR buka dari mobile

3. Storytelling kamu jadi lebih kuat

Kamu bisa “mengontrol alur”:

Hero
About
Skills
Projects
Experience
Contact

👉 Ini seperti pitch yang terarah, bukan loncat-loncat

4. Lebih cepat & simpel (important untuk fresh graduate)
Tidak perlu banyak routing
Fokus ke kualitas konten, bukan kompleksitas web
❌ Kekurangan Single Page (dan cara mengatasinya)
❌ Bisa terasa “panjang”

👉 Solusi:

Gunakan:
Section jelas
Divider
Sticky navbar (scroll ke section)
❌ Project bisa kurang detail

👉 Solusi:

Tambahkan:
“View Case Study” (modal / expandable)
atau link ke GitHub / PDF
🔀 Kapan pakai Multi-page?

Multi-page cocok kalau:

Kamu sudah punya banyak project kompleks
Kamu mau jadi specialist UI/UX atau ML engineer
Tiap project butuh deep explanation

👉 Tapi untuk kamu sekarang:
❌ Overkill
❌ Bisa bikin recruiter capek
